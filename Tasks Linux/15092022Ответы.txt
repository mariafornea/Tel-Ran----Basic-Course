1. Файл gitignore предназначен для скрытия от Git папок и файлов, это могут быть временные папки и файлы, те, которые содержат пароль или ту информацию, которая пока что не нужна, но и удалять безвозвратно не хочется.
2. Команда git remote rm origin используется для удаления репозитория
3. Конфликты слияний могут возникнуть в случае, когда один и тот же сегмент изменяют сразу несколько разработчиков, но при условии, что они работают на одной ветке. Так же может возникнуть конфликт, когда один человек пытается внести какие-то изменения в файл, а другой в это же время пытается этот файл удалить. Кроме этого конфликт может быть вызван и одним разработчиком. К примеру мы внесли изменения в файл в одной ветке, затем переключаемся на другую и вносим те же изменения. При слияние это может вызвать конфликт.
4. Если мы находимся, к примеру, на ветке test и перед слиянием хотим посмотреть какие изменения будут внесены в основную ветку master, нам необходимо переключиться на ветку master и только потом сравнивать: git checkout master; git diff ..test
5. Для удаления локальной ветки:
   git branch -d test     #предупредит нас перед удалением, если мы забыли про слияние (более безопасно удаление)
   git branch -D test     #удалит раз и навсегда, без возможности отмены 
   Для удаленной ветки:
   git push origin --delete master
6. Чтобы сделать файл неотслеживаемым, нужно удалить его из индекса:
   git rm --cached file
   git commit -am 'ignore file'
