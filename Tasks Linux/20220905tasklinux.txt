1. Ветка git это указатель на один из коммитов, как правило, на последний коммит. С каждым новым коммитом, этот указатель будет автоматически перемещаться. По умолчанию, основная ветка - master, но ей можно дать любое другое имя при создании.
2. SSH-ключи это безопасность, то есть используются для безопасного подключения к серверу. Если говорить о github, они нужны для того, чтобы загружая файлы в репозиторий, не вводить каждый раз имя пользователя и пароль.
3. Сервис github pages позволяет бесплатно размещать статические веб-сайты, не обращаясь к хостингам. Это своего рода инструмент для верстальщика, позволяющий верстать страницы, импортировать изображения, использовать привычные языки программирования. 


4. Создание ветки - git branch master
   Удаление ветки - git branch -D master
5. Переключение между ветками - git checkout master
6. Загрузка ветки на github - git push -u origin master
   Удаление ветки с github - git push origin --delete master
   
   
7. Для публикации статического веб-сайта на github:
    *Создаем аккаунт, если он есть - входим.
	*Создаем репозиторий
	*Загружаем все необходимые файлы в созданный репозиторий
	*Захоим в настройки репозитория, находим вкладку pages, в ней указываем в качестве источника файлов ветку main и директорию root, сохраняем.
	*Теперь сайт опубликован и доступен по ссылке account-name.github.io/repo-name
