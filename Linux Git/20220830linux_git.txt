1. Ветвление это способ немного отклониться в сторону от основного проекта, и продолжать работу, не затрагивая его. Ветка это своего рода указатель
на определенный коммит. По умолчанию, основная ветка - master. Она всегда будет указывать на последний коммит. Как только мы будем создавать новы
коммиты, указатель основной ветки также будет перемещаться. Имя основной ветки можно изменить, но по умолчанию после команды git init, ее имя
будет master. Создавая новую ветку, мы создаем новый указатель, который будет перемещаться между коммитами. 
2. Команды:
   * git branch test   // создание новой ветки
   * git checkout test   // переключение на новую ветку
   * git checkout -b test   // создание + переключение на новую ветку
   * git branch -v  test    //  просмотр последнего коммита на ветке
   * git branch -D test    //   удаление ветки
   * git push origin --delete test // еще один способ удалить ветку
